## 🔴 Maradó Kritikus Problémák

### 1. `PlaybackService` — God Object (521 sor, 5+ felelősség)

**Fájl:** `app/application/media/playback_service.py`

> ⚠️ Ez az eredeti auditból kimaradt, de **a legnagyobb God Object a kódbázisban**.

**Tartalmaz:**
- Média lejátszó detektálás (VLC/MPC-HC) → platform-specifikus infra logika
- Fájl indítás subprocess-szel → infra
- Playback monitoring (HTTP polling VLC/MPC API-khoz) → infra
- Watch history CRUD → users domain
- Resume position tracking → users domain
- File explorer megnyitás (reveal/open) → infra

**Javasolt szétdarabolás:**
- `infrastructure/playback/player_detector.py` — VLC/MPC-HC felderítés + indítás
- `infrastructure/playback/playback_monitor.py` — HTTP polling logic
- `application/media/playback_service.py` — orchestration (kis méretű)
- Watch history CRUD maradhat, de a `UserOverride` direkt manipulálás helyett port kéne

---

### 2. `RecommendationsService` — 405 sor, 8 domain importálva

**Fájl:** `app/application/recommendations/recommendations_service.py`

**Cross-domain importok:**
- `domains/users/models` (User, CustomList, CustomListItem)
- `domains/library/models` (MediaItem, Library, ExtraFile)
- `domains/metadata/models` (MetadataMatch, MetadataLocalization)
- `domains/settings/models` (UserSetting, SystemSetting)
- `domains/media_assets/services` (ImageProcessingService)
- `domains/tasks` (task_manager)
- `infrastructure/settings` (formatter_config_adapter)

**Probléma:** Application service-nek jogos több domain-ból olvasnia, de ez a service:
- Saját maga resolve-ol settings-et ahelyett hogy `SettingsPort`-ot használna (a port már létezik!)
- `ImageProcessingService()` közvetlenül példányosítja (nincs DI)
- `_call_api` privát metódust hív a TMDB scraper-en — leaking abstraction

---

### 3. `ScannerService` — 477 sor, mixed concerns

**Fájl:** `app/application/media/scanner_service.py`

**Probléma:**
- Global mutable `scan_status` dict thread lock-kal — ez egy implicit singleton state, amit nehéz tesztelni
- `get_history()` metódus itt van, de ez a history domain felelőssége
- Direkt import: `from app.infrastructure.scrapers.scan_resolver import ScanResolver` — infra direkt import application layer-ben

### 14. Dict-alapú return típusok — nincs típusbiztonság

A legtöbb service továbbra is `Dict[str, Any]`-t vagy `dict`-et ad vissza. Nincs Pydantic response model.

Egyes service-ek `{"error": "..."}` dict-et adnak, mások `JSONResponse`-t, megint mások `DomainException`-t dobnak. A `DomainException` handler létezik `main.py`-ben, de alig van használva.
