# conflux-start

Re-orient for the cheapest possible tokens. **Do not scan the repo.**

0. **If `/HOT-START.md` exists, read it first** — the one-paste resume (the Q16 "Hot Start Prompt",
   regenerated by the Loop each session-end). It alone usually tells you where we are + the single next
   action; if it's present and fresh you can often report from it directly and skip the rest.
   **A freshly-initialized host project has no HOT-START.md — that is normal, not an error.** Don't
   report a failed read; just continue with step 1.
1. Read `/AGENT-HQ.md` — compass, infra, conventions, and both legs' (Wex + Quas) load-bearing decisions.
2. Read `/PHASING-HANDOVER.md` — the open handover rows are exactly where the last session stopped.
3. *(optional, cheap)* recall relevant memory for the area you're resuming:
   `node "E:/projects/Lens-Levi/wex/grimoire/grimoire.mjs" recall "<area>" --limit 5`
4. Report in **≤ 8 lines**: where we are · the single next step (newest handover row, else current
   BUILD-TRACKER `🔨 wip`) · any `blocked` / `Loc:N` flags.
5. **Stop.** Wait for the user to confirm before doing work.

> Token rule: this skill reads two files (plus one grep-like recall). It must **never** trigger a
> full-repo read — that's the whole point of the handover.
